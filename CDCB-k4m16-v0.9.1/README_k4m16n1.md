# CDCB-k4m16-v0.9.1 代码说明文档

## 一、项目概述

本仓库为**基于固定偏置权重的可控动态多码本加密（CDCB）**C语言嵌入式实现，对应发明专利：
申请号：`202610926538.8`
标准配置：码本数量 k=4，单码本符号空间 m=16，半字节4bit分组加密；适配8位/32位MCU，无浮点运算，仅依靠数组置换与区间概率调度实现抗频率分析。

版本：v0.9.1 定稿版
文件清单：

1. `cdcb_k4m16.h`：结构体定义 + 全部API函数声明
2. `cdcb_k4m16.c`：码本生成、偏置区间计算、加解密核心实现
3. `main.c`：双端加解密自测用例

## 二、核心数据结构说明

### 结构体 `cdcb_k4m16_t`

```c
typedef struct {
    uint8_t  cdcb_key[32];        // 32字节主密钥，用于Fisher-Yates洗牌生成4组独立动态码本
    uint16_t base_codes[4];        // 4个旋转不等价基码，密文自带码本标识，无需额外同步字段
    uint8_t  k;                    // 固定等于4，4组并行码本
    uint8_t  m;                    // 固定等于16，每组码本包含0~16共16个半字节符号
    uint8_t  bias[16][4];          // 【核心创新】16个明文符号对应4个码本的分段区间边界，实现非均匀选码概率
    uint8_t  codebook[4][16];      // 正向置换码本：明文→密文偏移
    uint8_t  codebook_inv[4][16];  // 反向映射表，解密O(1)查表，无需遍历
} cdcb_k4m16_t;
```

1. `bias[16][4]`：对应专利中**固定偏置权重**，把明文频率拆分为高低双峰分布，消除统计特征；
2. 正向+反向双码本：加密随机置换，解密直接查表，满足嵌入式低时延需求；
3. `base_codes` 旋转不等价码：密文旋转还原基码，自动识别所属码本，不额外传输码本编号。

## 三、全局接口总览（memcpy风格调用，上层极简）

### 1. 密钥上下文更新（最核心入口）

```c
uint8_t cdcb_key_k4m16_update(
    cdcb_k4m16_t *key,
    const uint8_t cdcb_key[32],    // 32字节主密钥，传NULL则保留原有密钥
    const uint16_t kid[4],         // 4组基码，传NULL则不更新基码
    const uint32_t knowledge[16], // 明文先验频率统计（优先级别最高）
    const uint8_t *plaintext,      // 明文样本，用于实时统计频率
    const uint32_t plain_len
);
```

### 三层优先级自动生成偏置权重（对应专利S3步骤）

1. 传入`knowledge[]`：使用外部先验频率，生成双峰极化分割区间；
2. 无先验知识、传入明文缓冲区：自动统计明文符号出现次数，动态计算bias分割点；
3. 无任何统计数据：随机生成4组分割区间，保证多码本选取概率非均匀。

### 码本生成逻辑

以32字节密钥为随机种子，对4组码本分别执行Fisher-Yates随机洗牌，各组码本相互独立；单个码本泄露不会破坏整体安全性。

---

### 2. 基础单符号加解密（半字节4bit）

```c
// 加密：4bit明文 → 16bit带旋转基码的密文标签
uint16_t cdcb_encrypt_nibble(const uint8_t plain_4bit, void *k);

// 解密：16bit密文标签 → 还原4bit原始明文
uint8_t cdcb_decrypt_nibble(const uint16_t cipher,void *k);
```

### 加密流程

1. 根据当前明文符号的`bias`区间，随机选取4组码本之一；
2. 查找正向码本得到偏移量；
3. 对基码执行循环旋转，生成最终16位密文。

### 解密流程

1. 遍历旋转值，还原出原始基码，自动匹配对应的码本组；
2. 取出旋转偏移，查询反向映射表，O(1)得到明文符号。

> 关键特性：双端仅同步密钥+4个基码，密文自身携带码本ID，不需要额外同步通道。
> 

---

### 3. 字节流批量加解密（业务层直接调用）

```c
// 字节数组加密：1字节明文 → 4字节密文（高低两个半字节分别加密）
uint8_t cdcb_encrypt(void *key,
                    const uint8_t *plaintext,
                    const uint32_t plain_len,
                    uint8_t *ciphertext,
                    uint32_t *cipher_len);

// 字节数组解密
uint8_t cdcb_decrypt(void *key,
                    uint8_t *plaintext,
                    uint32_t *plain_len,
                    const uint8_t *ciphertext,
                    const uint32_t cipher_len);
```

- 输入任意长度字节流，自动拆分为高4bit、低4bit两个半字节独立加密；
- 输出密文长度严格等于 `明文长度 × 4`；
- 返回值：`0x00=正常`，`0x01=参数为空/越界异常`。

### 4. 基码旋转工具函数（自识别码本核心）

```c
// 对基码执行循环左移，生成密文标签
uint16_t CDCB_kid_rotate(uint16_t code, uint8_t n);

// 遍历所有旋转，还原原始基码，并输出旋转偏移量
uint16_t CDCB_kid_to_base(uint16_t kid, uint8_t* offset);
```

功能：任意旋转后的密文标签，均可唯一还原为原始不等价基码，实现无通信自动码本匹配。

## 四、调用流程（标准使用步骤）

1. 定义上下文结构体：`cdcb_k4m16_t key_ctx;`
2. 调用`cdcb_key_k4m16_update()`一次性完成：
    - 洗牌生成4组动态码本
    - 生成正向+反向映射表
    - 计算16个符号对应的4组偏置区间
3. 调用`cdcb_encrypt()`完成明文批量加密；
4. 接收方调用`cdcb_decrypt()`还原原始数据。

### 最简调用示例

```c
cdcb_k4m16_t ctx;
// 初始化码本与偏置（无先验统计，仅传入密钥与基码）
cdcb_key_k4m16_update(&ctx, key_buf, kid_buf, NULL, NULL, 0);

uint8_t cipher[64];
uint32_t c_len;
cdcb_encrypt(&ctx, "test", 4, cipher, &c_len);
```

## 五、核心代码模块拆分（对应专利创新点）

| 代码模块 | 对应专利技术方案 |
| --- | --- |
| `cdcb_do_bias()` | 双峰极化权重分割、非均匀固定偏置权重分配 |
| 4组独立Fisher-Yates洗牌码本 | 多组独立动态置换，乘积级攻击复杂度 |
| `CDCB_kid_to_base()` 旋转还原基码 | 旋转不等价码池，密文自标识码本，零同步开销 |
| 正向+反向双码本 | O(1)查表解密，适配低端MCU |
| 区间随机选码逻辑 | 打破明文与密文频率对应关系，抵御AI频率推理攻击 |

## 六、编译与平台适配说明

1. 纯标准C代码，仅依赖`stdint.h`、`stdbool.h`，无第三方库；
2. 随机数源：默认使用`rand()`，嵌入式项目可替换为硬件真随机数发生器；
3. 内存占用：
    - 结构体总RAM占用：不足256字节；
    - 无需Flash缓存，运行时实时生成码本；
4. 兼容性：C51、STM32、ESP32、Linux均可直接编译运行。

## 七、返回码规范

| 返回值 | 含义 |
| --- | --- |
| 0x00 | 执行成功 |
| 0x01 | 空指针、参数越界、缓冲区异常 |
